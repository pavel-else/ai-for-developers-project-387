# Сущности

## Owner

Предопределённый владелец календаря. Одна запись, создаётся при инициализации приложения.

- id — уникальный идентификатор
- name — имя владельца
- slug — публичный идентификатор (используется в `GET /{slug}/slots`)
- email — контактный email

## EventType

Тип события, который создаёт владелец календаря. Гость выбирает его перед бронированием.

- id — уникальный идентификатор
- ownerId — владелец
- name — название
- description — описание
- duration — длительность в минутах

Начальные (seed): "15 минут" (duration: 15), "30 минут" (duration: 30).

## Slot

Окно доступности, которое создаёт владелец.

- id — уникальный идентификатор
- ownerId — владелец
- startTime — начало окна
- endTime — конец окна

В одном Slot может быть несколько Booking.
Удаление слота блокировано, пока есть активные брони.

## Booking

Факт бронирования, созданный гостем.

- id — уникальный идентификатор
- slotId — в каком слоте
- eventTypeId — выбранный тип события
- startTime — начало встречи (≥ slot.startTime)
- endTime — конец встречи (≤ slot.endTime)
- bookerName — имя гостя
- bookerEmail — email гостя
- status — active / cancelled

Продолжительность встречи — из EventType (eventTypeId → duration).
На одно и то же время не может быть двух активных бронирований (проверка при создании, независимо от slotId и eventTypeId).
