# Пользовательские сценарии

## 1. Гость — запись на встречу

**Цель:** Гость выбирает тип события и бронирует свободное время.

**Предусловия:** Владелец создал хотя бы один тип события и хотя бы один слот доступности.

**Шаги:**
1. Гость открывает главную страницу (`/`)
2. Система показывает список типов событий (название, описание, длительность)
3. Гость выбирает подходящий тип события, нажимает «Select»
4. Система перенаправляет на страницу бронирования (`/book?eventTypeId=X&duration=Y`)
5. Система загружает доступные временные окна на ближайшие 14 дней
6. Гость выбирает время из сетки доступных слотов
7. Появляется форма ввода имени и email
8. Гость заполняет форму и нажимает «Confirm Booking»
9. Система создаёт бронь и показывает экран подтверждения с деталями встречи
10. Гость видит кнопки: «View My Bookings» и «Back to Home»

**Постусловия:** Бронь создана со статусом `active`. Выбранное время больше не отображается как доступное для других гостей.

**Исключения:**
- Нет доступных слотов на выбранную длительность → сообщение «No available slots for this duration»
- Конфликт времени при создании (другой гость забронировал то же время) → ошибка, предложить выбрать другой слот
- Невалидные данные формы (пустое имя, некорректный email) → валидация на клиенте, форма не отправляется

---

## 2. Гость — просмотр и отмена своих броней

**Цель:** Гость смотрит свои активные/отменённые брони по email и может отменить предстоящую встречу.

**Предусловия:** Гость ранее создал хотя бы одну бронь.

**Шаги:**
1. Гость открывает страницу «My Bookings» (`/bookings`)
2. Гость вводит свой email в поле и нажимает «Search»
3. Система загружает список всех броней, привязанных к этому email
4. Каждая бронь отображается в карточке: дата, время, статус, имя гостя
5. Гость нажимает «Cancel» на активной брони
6. Система запрашивает подтверждение отмены
7. Гость подтверждает отмену
8. Статус брони меняется на `cancelled`, карточка обновляется

**Постусловия:** Бронь отменена (статус `cancelled`). Освободившееся время становится доступным для новых броней.

**Исключения:**
- Введён невалидный email → сообщение об ошибке
- По email не найдено ни одной брони → пустой список с сообщением
- Бронь уже отменена → кнопка «Cancel» неактивна, показан статус «Cancelled»

## 3. Админ — управление типами событий

**Цель:** Владелец создаёт, редактирует и удаляет типы событий (длительность встреч, которые может выбрать гость).

**Предусловия:** Нет.

**Шаги:**
1. Админ открывает страницу «Admin → Event Types» (`/admin/event-types`)
2. Система показывает список всех типов событий в карточках
3. **Создание:** админ нажимает «Create Event Type», заполняет название, описание и длительность (1–480 мин), нажимает «Save»
4. **Редактирование:** админ нажимает «Edit» на карточке, меняет поля в диалоге, нажимает «Save»
5. **Удаление:** админ нажимает «Delete» на карточке, подтверждает удаление

**Постусловия:** Тип события создан/изменён/удалён. При удалении связанные брони не затрагиваются (остаются с eventTypeId удалённого типа).

**Исключения:**
- Пустое название → валидация, форма не отправляется
- Длительность вне диапазона 1–480 → валидация
- Удаление несуществующего типа → ошибка 404

---

## 4. Админ — управление слотами доступности

**Цель:** Владелец указывает временные окна, в которые гости могут бронировать встречи.

**Предусловия:** Нет.

**Шаги:**
1. Админ открывает страницу «Admin → Slots» (`/admin/slots`)
2. Система показывает список всех слотов с их бронями
3. **Создание:** админ нажимает «Create Slot», выбирает дату и время начала/конца, нажимает «Save»
4. **Удаление:** админ нажимает «Delete» на слоте, подтверждает удаление

**Постусловия:** Слот создан или удалён. Внутри слота система автоматически находит свободные «окна» с учётом уже существующих броней.

**Исключения:**
- Дата начала позже даты конца → валидация на клиенте
- Удаление слота с активными бронями → ошибка 409, слот не удалён
- Удаление несуществующего слота → ошибка 404

---

## 5. Админ — просмотр и отмена броней

**Цель:** Владелец смотрит все предстоящие встречи и может отменить любую.

**Предусловия:** Есть хотя бы одна активная бронь.

**Шаги:**
1. Админ открывает страницу «Admin → Bookings» (`/admin/bookings`)
2. Система показывает список предстоящих активных встреч, отсортированных по дате
3. Каждая бронь отображается в карточке: дата, время, имя и email гостя, тип события, статус
4. Админ нажимает «Cancel» на брони
5. Система запрашивает подтверждение
6. Админ подтверждает отмену
7. Статус брони меняется на `cancelled`

**Постусловия:** Бронь отменена. Освободившееся время становится доступным для новых броней.

**Исключения:**
- Нет предстоящих броней → пустой список с сообщением
- Бронь уже отменена → кнопка «Cancel» неактивна
- Отмена несуществующей брони → ошибка 404
